# Desktop Navigation

Use this page as a starting point for moving around a Linux desktop after install. Exact shortcut defaults can vary by desktop environment, distro version, keyboard layout, and user settings, so confirm the current bindings in the desktop's keyboard shortcut settings.

## Common Concepts

- App launcher: opens apps by name and often searches settings, files, or commands
- Overview: shows open windows, workspaces, and sometimes the app grid
- Window switcher: moves between open windows
- Workspace switcher: moves between virtual desktops
- Tiling or snapping: places windows into screen regions
- Lock shortcut: locks the active session

## Desktop Families

- GNOME-family desktops commonly center navigation around the overview, app search, workspaces, and keyboard shortcuts configured in Settings.
- KDE Plasma desktops commonly center navigation around KRunner, the application launcher, window switching, virtual desktops, and highly configurable shortcuts.
- COSMIC centers navigation around its launcher, app library, workspaces, tiling controls, panels, and configurable keyboard shortcuts.
- Hyprland and other tiling window managers are usually keyboard-first and depend heavily on the distro or dotfiles that define the keybindings.
- Gaming Mode or console-like sessions may hide the normal desktop until the user switches back to a desktop session.

## First Checks After Install

- Open the keyboard shortcut settings and review the launcher, terminal, window switching, workspace movement, lock screen, screenshot, and tiling bindings.
- Learn the desktop's app launcher first; it is usually the fastest way to open settings, terminal, Steam, browser, and files.
- If the distro ships multiple desktop choices, use the selected desktop's page rather than assuming another edition has the same shortcuts.
